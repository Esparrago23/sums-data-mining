# Subcomponente A — OCR estructurado de cédulas PDF
#
# Entorno objetivo: Linux ARM64 / x86_64 dentro de Docker (Raspberry Pi 4 / Ubuntu Server).
#
# ─── Dependencias originales (pipeline de checkboxes, números y preprocesamiento) ───
#   numpy          : matrices y cálculo de features visuales
#   opencv-python-headless : binarización, contornos, HoughLines (sin GUI, apto servidor)
#   pymupdf        : renderizado de PDFs a PNG (fallback sin Poppler)
#   pytesseract    : OCR de texto manuscrito — FALLBACK si PaddleOCR no está disponible
#   scikit-learn   : modelos KNN / LogisticRegression locales
#
# ─── Motor principal de texto: PaddleOCR ─────────────────────────────────────────────
#   paddlepaddle   : framework DL backend de PaddleOCR (CPU-only en ARM64)
#                    ARM64 wheel: ver instrucciones en docker/Dockerfile.ocr
#   paddleocr      : API de alto nivel para detección + reconocimiento (PP-OCRv4)
#
# ─── Fuzzy matching ──────────────────────────────────────────────────────────────────
#   rapidfuzz      : similitud de cadenas (WRatio/partial_ratio) para corrección OCR
#
# ─── Dependencia del sistema (para pytesseract y renderizado PDF) ─────────────────────
#   Ubuntu/Debian: sudo apt install poppler-utils tesseract-ocr tesseract-ocr-spa
#   (en Docker: ver docker/Dockerfile.ocr para la imagen ARM64 completa)
